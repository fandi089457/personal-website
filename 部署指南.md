# 🚀 GitHub 自動化部署指南

## 📋 準備工作

### 1. 創建 GitHub Personal Access Token

**步驟：**
1. 登入 GitHub
2. 點擊右上頭像 → Settings
3. 左側選單 → Developer settings
4. 點擊 Personal access tokens → Tokens (classic)
5. 點擊 Generate new token
6. 設定權限：
   - ✅ `repo` (完整倉庫權限)
   - ✅ `workflow` (GitHub Actions 權限)
7. 複製生成的 Token (只顯示一次！)

### 2. 安裝 Python 環境

```bash
# 確認 Python 已安裝
python --version

# 安裝依賴
pip install -r requirements.txt
```

## 🎯 執行自動部署

### 方法一：運行腳本
```bash
python deploy.py
```

### 方法二：手動輸入資訊
腳本會詢問：
- 🔑 GitHub Token
- 👤 GitHub 用戶名  
- 📁 倉庫名稱

## ✅ 自動化完成的功能

1. **創建 GitHub 倉庫**
2. **上傳所有網站文件**
3. **設置 GitHub Actions**
4. **啟用 GitHub Pages**
5. **配置自動部署**

## 🌐 結果

部署完成後，您的網站將在：
```
https://[您的用戶名].github.io/[倉庫名稱]/
```

## 🔄 未來自動化

**之後只需：**
1. 修改網站文件
2. `git push` 到 GitHub
3. 自動部署完成！

## 🛠️ 故障排除

### Token 權限不足
- 確保勾選 `repo` 和 `workflow` 權限
- 重新生成 Token

### 倉庫已存在
- 腳本會自動處理
- 或手動刪除舊倉庫

### 部署失敗
- 檢查網路連線
- 確認 Token 有效期
- 查看錯誤訊息
